Bases de Datos

Eje de la Prueba Funcional

Información del Tema

El propósito principal del cargo de Profesional Especializado es orientar la incorporación de nuevas tecnologías, para la construcción de soluciones innovadoras, que aporten mejoras en el análisis y procesamiento de información. Esto se logra, en parte, a través de la gestión y optimización de las **bases de datos**: 3].

El manual de funciones especifica una serie de conocimientos y habilidades clave para este tema, como la **metodología para el modelamiento de base de datos** y el manejo de **herramientas de administración de bases de datos**: 219].

Conceptos Fundamentales y Tipos

Una **base de datos** es un sistema organizado para almacenar, gestionar y recuperar información de manera eficiente y segura: 3]. El **SGBD** (Sistema Gestor de Bases de Datos) es el software que controla el acceso y garantiza la integridad de los datos: 3].

Tipos de Bases de Datos:

  • **Relacionales (SQL):** Organizan la información en tablas. Son ideales para datos estructurados y transaccionales, ya que garantizan las propiedades **ACID** (Atomicidad, Consistencia, Aislamiento, Durabilidad): 2, 3]. Ejemplos: PostgreSQL, MySQL.
  • **No Relacionales (NoSQL):** Son más flexibles y escalables, adaptándose a estructuras no rígidas. Se usan para datos semiestructurados o no estructurados: 2]. Ejemplos: MongoDB, Redis.

Modelado, Diseño y SQL

Modelado Relacional:

  • **Normalización:** Proceso para eliminar redundancias y anomalías de actualización, y se recomienda ir hasta la 3FN o BCNF: 1, 2].
  • **Claves:** Se usan claves primarias (PK), únicas (UK) y foráneas (FK) para definir restricciones e integridad: 1].

SQL Avanzado:

El dominio del lenguaje SQL es fundamental: 3]. Las pruebas suelen incluir:

  • **Joins:** Conocer y usar diferentes tipos como `INNER`, `LEFT`, `RIGHT` y `FULL JOIN`: 1].
  • **Funciones de Ventana:** Permiten realizar cálculos sobre un conjunto de filas relacionadas con la fila actual (`PARTITION BY`, `ORDER BY`, `lag/lead`): 1, 2].
  • **CTEs (Common Table Expressions):** Consultas complejas que se definen en una estructura temporal y facilitan la lectura del código: 2].
  • **Vistas y Vistas Materializadas:** Las vistas materializadas persisten el resultado de una consulta y deben ser actualizadas (`REFRESH`) para tener datos recientes: 1].

Rendimiento y Mantenimiento

Para mejorar el rendimiento y la escalabilidad, se usan diversas técnicas:

  • **Índices:** Aceleran las búsquedas y consultas. Los tipos más comunes son B-tree, GIN y BRIN, cada uno con un uso específico: 1, 2].
  • **Análisis y Optimización:** El comando `EXPLAIN ANALYZE` permite ver el plan de ejecución de una consulta para optimizarla: 1, 2].
  • **Particionamiento:** Divide tablas grandes en partes más pequeñas para mejorar la velocidad: 1].
  • **Mantenimiento:** Herramientas como `VACUUM/ANALYZE` en PostgreSQL son esenciales para limpiar datos obsoletos y mantener la integridad: 1].

Seguridad y Normativa en Colombia

La seguridad y el cumplimiento normativo son críticos en el sector público: 3].

  • **Seguridad:** Incluye el control de acceso, el cifrado de datos (en tránsito y en reposo) y el uso de roles y privilegios granulares: 1, 3].
  • **Normativa:** Es vital conocer la **Ley 1581 de 2012** (Habeas Data) para la protección de datos personales y el **Decreto 1008 de 2018** sobre gobierno de datos en entidades públicas: 2, 3].

Tips de Examen:

Enfócate en software libre como **PostgreSQL**, su relación con el **Gobierno de Datos** y la aplicabilidad de los conceptos a escenarios prácticos del sector público. Repasa la normativa colombiana y las diferencias entre sistemas relacionales y NoSQL. Presta atención a las trampas de los valores `NULL` en SQL y a la lógica de tres valores (TRUE/FALSE/UNKNOWN): 1].

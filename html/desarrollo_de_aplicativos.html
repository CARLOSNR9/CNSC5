Desarrollo de Aplicativos

Eje de la Prueba Funcional

Información del Tema

El desarrollo de aplicativos consiste en diseñar, construir, probar, implementar y mantener software que atienda necesidades específicas, ya sea para escritorio, web, móvil o sistemas empresariales. Es un proceso multidisciplinario que combina programación, ingeniería de software, experiencia de usuario, e integración con bases de datos y servicios externos.

El propósito principal del cargo de Profesional Especializado es orientar la incorporación de nuevas tecnologías para la construcción de soluciones innovadoras, que aporten mejoras en el análisis y procesamiento de información. Esto implica participar activamente en el desarrollo y la gestión de proyectos de TI.

Conceptos Clave y Proceso de Desarrollo

El ciclo de vida del software (SDLC) puede seguir diferentes modelos:

  • **Modelos Clásicos:** Cascada (útil para requisitos estables).
  • **Metodologías Ágiles:** Scrum, Kanban, SAFe.
  • **DevOps/DevSecOps:** Una cultura que integra desarrollo, operaciones y seguridad, con prácticas como la entrega continua (CI/CD).
  • **Plataform Engineering:** Tendencia que busca optimizar los flujos de desarrollo.

La **Arquitectura de Software** es fundamental. Conceptos como la **Arquitectura Hexagonal**, que aísla las reglas de negocio de la infraestructura (bases de datos, UI), y el **12-Factor App**, una metodología para construir aplicaciones modernas, suelen ser evaluados.

Tecnologías, Arquitecturas y Herramientas

Arquitecturas Modernas:

  • **Microservicios:** Descomponer una aplicación en servicios independientes para mejorar la escalabilidad y el despliegue autónomo.
  • **Serverless:** Funciones como servicio (FaaS) que escalan automáticamente, como AWS Lambda o Azure Functions.
  • **Arquitectura de Eventos (EDA):** Manejo de flujos de datos asíncronos.

Tecnologías y Frameworks:

Se evalúa el conocimiento de tecnologías populares en el sector público, priorizando el software libre. Lenguajes como **Python**, **Java** y **Node.js** con frameworks como **Django** o **Spring Boot** son comunes. En el frontend, destacan **React**, **Angular** y **Vue**.

DevOps y CI/CD:

La integración continua (CI) y la entrega continua (CD) son prácticas esenciales. Herramientas como **Docker** y **Kubernetes** son el estándar para empaquetar y orquestar aplicaciones. La **observabilidad** (con métricas, logs y trazas) es clave para monitorear el rendimiento en producción. Los 4 métricos **DORA** (frecuencia de despliegue, tiempo de entrega, tasa de fallas, tiempo de recuperación) miden el desempeño del equipo.

Seguridad, Calidad y Cumplimiento Normativo

La **calidad de software** se evalúa con estándares como **ISO/IEC 25010** que define atributos como fiabilidad, eficiencia, usabilidad y seguridad. La **seguridad** es un tema transversal (**DevSecOps**), y los referentes más comunes son el **OWASP Top 10** para riesgos web y el **OWASP API Security Top 10** para APIs.

Pruebas:

La pirámide de pruebas (muchas pruebas unitarias, menos de integración, pocas de extremo a extremo) es un modelo clave. Se usan herramientas como **Jest** o **Cypress**.

Normativa Colombiana:

Es importante conocer:

  • **Manual/Política de Gobierno Digital (MinTIC):** Articula la arquitectura y la gestión de proyectos de TI.
  • **Ley 1581 de 2012:** Protección de datos personales.
  • **Guía de Desarrollo de Software Seguro para entidades públicas (MinTIC):** Brinda lineamientos de seguridad.
  • **WCAG 2.2:** Estándar de accesibilidad web que es obligatorio en el sector público para asegurar la inclusión digital.

Funciones Clave Relacionadas

Las preguntas del simulacro se centrarán en la aplicación práctica de estas funciones:

  • **Trabajar** con los diferentes líderes de desarrollo en la construcción de soluciones flexibles, escalables y que se encuentren bajo los lineamientos de arquitectura.
  • **Promover** la implementación y gestión de nuevas tecnologías para innovar y proporcionar nuevas ideas de proyectos / productos.
  • **Diseñar y estructurar** modelos de datos para los requerimientos de desarrollo e intervención de los sistemas de información.
  • **Evaluar** la implementación de herramientas de software libre como primera opción de recursos técnicos.

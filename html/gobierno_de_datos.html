Gobierno de Datos

Eje de la Prueba Funcional

Información del Tema

El **Gobierno de Datos** es una disciplina directiva y un marco integral que define las políticas, procesos, estándares y roles para gestionar los datos como un activo estratégico en una organización. Su propósito es asegurar que los datos sean confiables, seguros, utilizables y cumplan con la normativa, potenciando la toma de decisiones y los servicios públicos. : 1, 2]

Marco y Estructura

Principios y Normas Clave (2025):

  • **DAMA-DMBOK:** Marco global de referencia que integra 11 funciones de gestión de datos, incluyendo la gobernanza, calidad, arquitectura y metadatos. : 1, 3]
  • **DCAM (EDM Council):** Modelo para evaluar la madurez del gobierno de datos, con una versión v3 anunciada para 2025. : 1, 3]
  • **FAIR:** Principios (Findable, Accessible, Interoperable, Reusable) clave para la reutilización de datos, especialmente en temas de datos abiertos. : 1]
  • **DCAT v3 (W3C, 2024):** Vocabulario estándar para catálogos de datos que asegura la interoperabilidad de metadatos. : 1]

Roles y Responsabilidades:

En el gobierno de datos, es fundamental la clara asignación de roles. Los más comunes son: : 1, 2, 3]

  • **Chief Data Officer (CDO):** Lidera la estrategia y define las políticas.
  • **Data Owner (Dueño de dato):** Responsable de negocio del dato, define las reglas y el acceso.
  • **Data Steward:** Custodio operativo encargado de la calidad, metadatos y linaje.
  • **Data Custodian/IT:** Se encarga del soporte técnico, la seguridad y el almacenamiento.

Marco Normativo Colombiano

El conocimiento de la normativa local es crucial para las pruebas de la CNSC. Los documentos que debes dominar son: : 1, 2, 3]

  • **Ley 1581 de 2012:** El régimen general para la protección de datos personales (Habeas Data).
  • **Ley 1712 de 2014:** La ley de Transparencia y Acceso a la Información Pública, que promueve los datos abiertos. : 1]
  • **Resolución MinTIC 1519 de 2020:** Establece estándares para la publicación de información y datos abiertos. : 1]
  • **MSPI (Modelo de Seguridad y Privacidad de la Información):** Un marco de seguridad actualizado por la Resolución 02277 del 3-jun-2025, alineado con estándares internacionales como ISO 27001. : 1, 2]
  • **Decreto 1008 de 2018:** Fija los lineamientos para la implementación del gobierno de datos en entidades públicas. : 2]

Procesos y Buenas Prácticas

  • **Gestión de Calidad:** Se rige por estándares como **ISO/IEC 25012** (modelo de calidad) y **ISO/IEC 25024** (métricas de medición). : 1]
  • **Gestión de Metadatos:** Implica la creación de un catálogo y un diccionario de datos, donde el **linaje** (trazabilidad de los datos) es un proceso crítico para la auditoría. : 1, 2]
  • **Seguridad y Privacidad:** Es un componente esencial que incluye la clasificación de datos, el control de acceso, y la aplicación de marcos como el MSPI del MinTIC. : 1, 2, 3]

Tips Clave para el Examen

  • **Distingue conceptos:** El gobierno de datos define la política y la estrategia, la **calidad de datos** mide la exactitud y la **gestión de datos** se encarga de las tareas operativas. : 3]
  • **Conecta con el cargo:** La función de “Gestionar los modelos de datos y sus estructuras, acordes al gobierno de datos” es un reflejo directo del propósito del cargo de Profesional Especializado. : 1]
  • **Familiarízate con los roles:** En un caso situacional, ten claro quién es el **Data Owner** (responsable de negocio) y quién el **Data Steward** (responsable operativo). : 1, 2]
  • **Usa la normativa:** Para justificar tus respuestas, cita la ley o el marco de referencia correcto (por ejemplo, **Ley 1581 de 2012** para privacidad o **Ley 1712 de 2014** para datos abiertos). : 1, 2]

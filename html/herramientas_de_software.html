Herramientas de Software

Eje de la Prueba Funcional

Información del Tema

El manual de funciones del cargo de Profesional Especializado, Código 2028, establece una función clave: **"Evaluar y valorar, la implementación de herramientas de software libre, como primera opción de recursos técnicos a utilizar en la formulación de los proyectos."**: 4]. Este eje se centra en el conocimiento del aspirante sobre las herramientas y tecnologías disponibles, con un énfasis particular en aquellas de código abierto.

Las herramientas de software son aplicaciones y plataformas que optimizan el desarrollo, gestión, mantenimiento, pruebas y despliegue de sistemas informáticos y procesos tecnológicos. Su conocimiento es crucial para optimizar recursos técnicos y económicos en los proyectos de la entidad: 1, 2, 3].

Categorías y Conocimientos Clave

Las preguntas de esta sección evaluarán tu comprensión en áreas como:

1. Herramientas de Desarrollo y DevOps:

  • **Control de Versiones (VCS):** El dominio de **Git** y flujos de trabajo como `pull request` es fundamental: 1].
  • **Automatización (CI/CD):** Se evalúa el uso de herramientas como **GitHub Actions**, **GitLab CI/CD** o **Jenkins** para automatizar la construcción, pruebas y despliegue: 1].
  • **Contenedores y Orquestación:** El conocimiento de **Docker** para empaquetar aplicaciones y **Kubernetes** para orquestar y escalar los despliegues es clave: 1].
  • **Seguridad (DevSecOps):** Se aplican herramientas como **SonarQube** para análisis de calidad estática, **Snyk** para escanear vulnerabilidades en dependencias y **Trivy** para escanear imágenes de contenedores: 1].

2. Herramientas de Datos e Integración:

  • **Bases de Datos:** Se prioriza el conocimiento de motores de código abierto como **PostgreSQL** y **MySQL**, junto con herramientas de gestión como **pgAdmin**: 2].
  • **Análisis y BI:** Herramientas de software libre para la visualización y análisis de datos como **Metabase**, **Apache Superset**, **R** o **Python**: 1, 2].
  • **Orquestación de Datos:** Herramientas como **Apache Airflow** para orquestar flujos de trabajo de datos y **dbt** (data build tool) para la transformación de datos: 1, 2].

3. Herramientas de Infraestructura y Cumplimiento:

  • **Infraestructura como Código (IaC):** Se evalúa el uso de herramientas como **Terraform** para provisionar infraestructura y **Ansible** para automatizar la configuración: 1, 2].
  • **Observabilidad:** El estándar **OpenTelemetry** es fundamental para instrumentar trazas, métricas y logs de manera neutral: 1].
  • **Cumplimiento Normativo:** Se evalúa el conocimiento de estándares de accesibilidad como **WCAG 2.2** : 1] y la **Resolución MinTIC 1519 de 2020** para publicación de datos abiertos: 1]. El **MSPI** de MinTIC es el marco de seguridad nacional de referencia: 1].

El aspirante debe demostrar su capacidad para seleccionar la herramienta adecuada para un propósito específico, priorizando soluciones de software libre en un contexto de servicio público: 4, 1, 2, 3].
